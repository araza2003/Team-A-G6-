Question 2.16

The head element is declared as follows in the XHTML 1.0 DTD:

<!ENTITY % head.misc "(script|style|meta|link|object)*">
<!ELEMENT head (%head.misc;,
    ((title, %head.misc;, (base, %head.misc;)?) |
     (base, %head.misc;, (title, %head.misc;))))>

(a) Describe in English what the head.misc entity represents.

(b) Describe in English what the content specification of the head element represents. For this part of the problem, just translate the content specification directly into English. Also, you can use head.misc in your description; that is, you do not need to directly mention script, style, etc., in your description.

(c) Now rewrite your description from the previous part of this problem more succinctly. That is, analyze the content specification and attempt to understand what it means, not just what it says literally. Once you understand the content specification, you should be able to describe it in English reasonably simply.

Solution:

Part (a): What does the head.misc entity represent?

The head.misc entity represents a group of elements that are commonly found in the head section of an XHTML document. Specifically, it includes the following elements:

  • script - Used for embedding JavaScript.

  • style - Used for embedding CSS styles.

  • meta - Provides metadata about the document, such as keywords or description.

  • link - Used to link to external resources like stylesheets.

  • object - Used to embed multimedia or other objects like Java applets.

In essence, head.misc is a collection of these elements that can appear in any order within the head section of a document.

Part (b): Content Specification of the head Element

The content specification of the head element is as follows:

It consists of:

  • A combination of head.misc (which includes script, style, meta, link, and object elements),

  • Followed by a title element,

  • Optionally, more head.misc elements, and

  • Then a base element (which can appear before or after title and head.misc).

The base element is optional but can appear either before or after the title and any head.misc elements.

Part (c): Succinct Description of the Content Specification

The content specification for the head element describes a flexible structure. The head section must contain:

  • A mix of metadata elements like script, style, meta, link, and object (as defined in head.misc),

  • Followed by a title element,

  • And optionally, a base element, which can appear before or after the title and other metadata elements.

In summary, the head element must contain metadata elements (as described by head.misc), followed by a title, and optionally a base element.
