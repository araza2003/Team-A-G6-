Exercise 2.13

What is wrong with the following markup?

<strong>
    <dl>
        <dt>SGML</dt>
            <dd>Standard Generalized Markup Language</dd>
        <dt>XML</dt>
            <dd>Extensible Markup Language</dd>
    </dl>
</strong>

Answer

The issue with this markup is that the <strong> element is applied incorrectly around the entire <dl> (definition list) element. The <strong> element is used to indicate that the enclosed text should be emphasized as important, but it should not be used to wrap structural elements such as <dl>, <dt>, or <dd>.

Here's the corrected markup:

<dl>
    <strong>
        <dt>SGML</dt>
            <dd>Standard Generalized Markup Language</dd>
    </strong>
    <strong>
        <dt>XML</dt>
            <dd>Extensible Markup Language</dd>
    </strong>
</dl>

Explanation:

The <strong> element should only be used to emphasize text within a block element, not around a group of elements that are meant to define a list or a dictionary entry. Therefore, the <strong> tag should be removed from around the entire <dl> element and added it only to SGML and XML.
