Exercise 2.8

What is wrong with the following markup?

// Java code example
if (a < b && c != d) {
    System.out.println("Time for donuts!");
}

Answer

The markup provided is incorrect because it uses Java code within HTML without proper escaping for special characters such as `<`, `>`, and `&`. In HTML, these characters must be replaced with HTML entities to avoid confusion with HTML tags and to ensure correct rendering in the browser.

The corrected version of the code would look like this:

<!-- Java code example -->
if (a < b && c != d) {
    System.out.println("Time for donuts!");
}
